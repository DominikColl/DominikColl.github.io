While researching I found out that a good way to organize my repo is by feature branching. This is done by creating a new branch for each new feature or bug fix. This makes it very organized and each issue will have its own branch where it was or is being worked on. I can implement this style in my revamp projects creating issues that need to be fixed in my application and creating feature branches to commit to when working on that feature/bug. Why is it important to write effective Epics & User Stories? It is important to write good user stories so that you can take a step back from the development mindset and think like the user that will be using your application. This helps with development because you realize the user will not think like you when using what you built. By doing this you make it clear what the purpose of the user is using your app and what they need it to do for them. Name at least one (1) benefit to using Flowcharts & Pseudo Code for prototyping new features before re-vamping a project. Explain. By creating a flow chart you are mapping out the process of an app or new feature and possibly finding errors in the process that you thought would work in your head. This has happened to me before where I think the steps I have in my head will work but there is a key piece missing when I develop it and I usually have to backtrack and rebuild a piece of the project. Psuedo Code is good because it makes you think out the process and possible bug or troubles you will have before writing any code.
